# Conversation Generation Pipeline

Generates synthetic annotated conversations for training the Layer 2 clinical marker classifiers.

## What It Does

This pipeline produces realistic patient-partner conversations with utterance-level clinical marker annotations and simulated paralinguistic features. Each conversation is generated by a large language model and then annotated in a separate pass, producing training data in the format the classifier expects.

## File Structure

| File | Purpose |
|------|---------|
| `main.py` | Entry point — configures the run and manages the generation loop |
| `generate_conversations_testing.py` | Core generation logic: transcript parsing, dataclasses, validation, quality scoring |
| `PromptGenerationLogik.py` | Prompt construction, diversity system, and annotation chunking |
| `JsonUtils.py` | JSON parsing, repair, and atomic file I/O |
| `StaticSeedData.py` | Contains the conversation seed information for diversity.py, not included in the project repo.
| `diversity.py` | Contains logic to calcluate diversified and evenly distributed parameters for conversation

## Diversity System

The pipeline includes a composable diversity system that prevents repetitive output across large batches. Instead of generating conversations from simple demographic labels, each conversation is built from independently sampled traits across multiple dimensions:

- **Patient characterization** — composed from five independent trait pools (communication style, vulnerability style, social orientation, conversational anchors, occupational background), producing 32,768 unique character combinations
- **Age calibration** — adjusts all patient traits for age-appropriate speech patterns, vocabulary, physical context, and generational norms across the 55–89 age range
- **Partner characterization** — combines role-specific guidelines with a conversational dynamic trait that controls how the partner handles silence, fills space, and responds to patient difficulty
- **Turn distribution** — computed from the clinical profile, partner dynamic, and patient communication style, producing conversations that range from partner-dominant to patient-dominant
- **Conversation structure** — 10 arc types (gradual emergence, triggered by topic, escalation, etc.) and 15 situational seeds that vary the narrative shape and opening conditions
- **Batch-aware tracking** — a running memory of what has been generated steers each new conversation away from overrepresented patterns

All trait selections use a least-used-first algorithm that maintains even distribution across the corpus.

## Output Format

Each conversation produces three files:

- `CONV-XXXX_transcript.txt` — the annotated conversation with timestamps and paralinguistic features
- `CONV-XXXX_annotation.json` — utterance-level marker labels (all 38 markers per utterance)
- `CONV-XXXX_meta.json` — configuration, timing, quality score, and marker counts

A `generation_report.json` summarizes the full batch: marker coverage, profile distribution, quality metrics, and underrepresented markers.

## Usage

```bash
python main.py --count 10 --output ./test_batch
```

## Configuration

Generation parameters (model, temperature, thinking level, token limits) are set in the main configuration. The diversity system is automatic — it tracks selections internally and requires no manual configuration beyond enabling it with a tracker instance.
